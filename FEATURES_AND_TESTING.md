# تأكيد الميزات ودليل التجربة

## 1. ما تم تنفيذه (تأكيد)

### إصلاح نقاط البانوراما (Hotspots)
- **تم:** في `edit_hotspots.html`:
  - حساب الإحداثيات من **مكان النقر** عبر `mouseEventToCoords(event)` (وليس من مركز الكاميرا).
  - إيقاف الدوران التلقائي أثناء وضع الإضافة (`stopAutoRotate`, `stopMovement`) واستئنافه بعد الإلغاء.
- **تم:** في `panorama360_bp.py`:
  - دوال `_normalize_yaw_deg` و `_clamp_pitch_deg`.
  - التحقق من `panorama_id`, `asset_id`, `pitch`, `yaw` في API إضافة النقطة وإرجاع 400 عند الخطأ.
  - تطبيع وقفل القيم قبل الحفظ في قاعدة البيانات.

*(ملاحظة: إذا كانت النقطة ما زالت "طائرة" عندك، جرّب على صورة 360 رسمية مثل bma-1.jpg أو تأكد أن النقر على نفس الـ container الذي يستخدمه Pannellum.)*

## 2. المفضلة + بانوراما 360 + البلاغ مع مرفق صورة

### المفضلة (حفظ أماكن/بانوراما)
- **موجودة وتعمل:**
  - من صفحة عرض البانوراما: زر **"المفضلة"** يضيف البانوراما الحالية إلى المفضلة (جلسة المستخدم).
  - عرض المفضلة: من القائمة **المفضلة** أو الرابط `/favorites` (يتطلب تسجيل دخول).
- **كيف تجربها:**
  1. سجّل دخول.
  2. اذهب إلى **البانوراما 360** → معرض البانوراما → اختر بانوراما → **عرض**.
  3. في صفحة العرض اضغط **"المفضلة"** (أيقونة القلب).
  4. اذهب إلى **المفضلة** من القائمة: يجب أن تظهر البانوراما المحفوظة، ويمكنك فتحها أو إزالتها.

### رفع البلاغ من النقاط (Hotspots) مع إرفاق صورة
- **موجودة وتعمل:**
  - في صفحة عرض البانوراما: اضغط على **نقطة حمراء** (hotspot) → تفتح نافذة التفاصيل.
  - من النافذة: **"بلاغ"** يفتح نموذج بلاغ (نوع المشكلة، ملاحظات، **إرفاق صورة/فيديو**).
  - الإرسال يستدعي `POST /panorama360/submit-report` (نموذج مع `FormData` وملف مرفق).
  - المرفق يُحفظ في `static/uploads/reports/` والبلاغ يُسجّل في جدول **Issue** مع `image_path`.
- **كيف تجربها:**
  1. افتح أي بانوراما 360 من **معرض البانوراما** → **عرض**.
  2. اضغط على إحدى النقاط الحمراء على المشهد.
  3. في نافذة التفاصيل اضغط **"بلاغ"**.
  4. اختر **نوع المشكلة**، اكتب **ملاحظات**، واختر **صورة** من جهازك (اختياري).
  5. اضغط **إرسال**.

### أين يظهر البلاغ بعد الإرسال؟
- البلاغ يُحفظ كسجل في جدول **Issue** (بلاغات النظام).
- **للمشرف/المسؤول:** قائمة **البلاغات** من القائمة أو الرابط `/issues`.
- **للمستخدمين حسب الصلاحيات:** قائمة **البلاغات المستلمة** من القائمة أو الرابط **`/received-issues`** (بلاغات معينة للمستخدم أو لمجموعته أو الموزعة عليه).
- تفاصيل البلاغ من `/issues/<issue_id>`؛ حقل **صورة/مرفق** يعرض المسار المحفوظ (مثل `uploads/reports/...`) إن وُجد.

## 3. الشات بوت والدردشة الصوتية والإرشاد المكاني

### ما هو مكتمل حالياً
- **صفحة الشات بوت:** `/trips-ai/chatbot` (من القائمة: الرحلات / المساعد الذكي أو الشات بوت).
- **نص → إرسال:** حقل نص + زر إرسال، والرد يأتي من الذكاء الاصطناعي (`advanced_ai_system.get_ai_response`).
- **التعرف على الصوت (من المتصفح فقط، بدون كتابة):**
  - زر المايكروفون 🎤 يفعّل **Web Speech API** (`SpeechRecognition`) باللغة `ar-SA`.
  - بعد التحدث، النص يُعرض في الحقل ويُرسل تلقائياً، ثم يُقرأ الرد صوتياً (TTS عربي).
- **الرد صوتاً:** بعد كل رد من البوت يتم استدعاء `speakText(...)` لقراءة جزء من الرد (حد ~250 حرف).
- **زر "إرشاد مكاني ذكي":** يطلب من المستخدم أن يقول أو يكتب المكان الذي يريد الوصول إليه (مثل "جمرة العقبة")؛ حالياً يقتصر على توجيه المستخدم لكتابة/قول الوجهة دون ربط بخريطة أو مسار فعلي.

### ما يمكن إكماله لاحقاً (بدون تنفيذ هنا)
- **ربط الإرشاد المكاني بخرائط/مسارات:** استخدام واجهة خرائط (مثل Google Maps أو بيانات داخلية) وإرجاع اتجاهات حقيقية من الذكاء الاصطناعي أو من خدمة توجيه.
- **استخدام نماذجك (Gemini / ChatGPT) للصوت بالكامل:** الواجهة جاهزة؛ الخلفية تستخدم بالفعل `AIModel` و`get_ai_response`. تأكد من إضافة نموذج في **إدارة نماذج الذكاء الاصطناعي** (`/trips-ai/ai-models`) وربط مفتاح API (Gemini/OpenAI) حتى تكون الردود من خدمتك.

### كيف تجرب الشات بوت والصوت
1. اذهب إلى **`/trips-ai/chatbot`**.
2. **بدون كتابة (صوت فقط):** اضغط زر المايكروفون 🎤 → تكلّم بالعربية → بعد انتهاء التعرف يُرسل النص تلقائياً ويُقرأ الرد صوتياً.
3. **نص:** اكتب في الحقل واضغط إرسال.
4. **الإرشاد المكاني (خرائط + مسارات):** اضغط **"إرشاد مكاني ذكي"** ثم اكتب أو قل الوجهة (مثل "المسجد الحرام"، "جمرة العقبة"، "منى"). ستظهر خريطة مصغّرة للموقع ورابط **"فتح الاتجاهات في الخريطة"** يفتح Google Maps مع المسار من موقعك (إن سمحت بالموقع) أو من مكة افتراضياً.

## 4. تقارير الذكاء الاصطناعي (Excel + تصور)

- **موجودة وتعمل:**
  - الصفحة: **`/ai-reports`** (تقارير الذكاء الاصطناعي).
  - ثلاثة أنواع: **بلاغات**، **قراءات كهربائية**، **أصول ومعدات**.
  - عند الاختيار يُولَّد ملف **Excel** (.xlsx) يتضمن:
    - ورقة بيانات (بلاغات / قراءات / أصول).
    - ورقة **إحصائيات** و**رسم بياني** (BarChart) لتوزيع الحالات.
    - ورقة **ملخص ذكي**: إن وُجد مفتاح Gemini في الطلب أو في الإعدادات، يُولَّد نص ملخص بالذكاء الاصطناعي؛ وإلا يُستخدم ملخص ثابت.
- **ملاحظة "خرائط ذهنية":** التصور الحالي هو **رسوم بيانية داخل Excel** (أعمدة). رسم خرائط ذهنية حقيقية (Mind Map) يتطلب إما إضافة مكتبة لرسم Mind Map في الويب أو تصدير بيانات لبرنامج خارجي؛ لم يُنفَّذ في الكود الحالي.

### كيف تجرب التقارير
1. سجّل دخول (يفضل صلاحية تسمح بالوصول لصفحة التقارير).
2. اذهب إلى **`/ai-reports`**.
3. اختر أحد الأنواع الثلاثة (بلاغات / قراءات / أصول) بالضغط على البطاقة المناسبة.
4. (اختياري) أدخل **مفتاح Gemini** في الحقل إن رغبت في ملخص ذكي داخل Excel.
5. سيُحمَّل ملف Excel تلقائياً؛ افتحه وتصفح الأوراق والرسم البياني وورقة "ملخص ذكي".

## 5. ربط الموقع بقاعدة بيانات خارجية (MySQL / PostgreSQL)

- **جاهز من ناحية الكود:**
  - التطبيق يستخدم **`SQLALCHEMY_DATABASE_URI`** من متغير البيئة **`DATABASE_URL`** (في `app.py`).
  - ملف **`.env.example`** يوضح أمثلة لـ SQLite (افتراضي)، MySQL، PostgreSQL، و SQL Server.
- **ما عليك فعله للربط:**
  1. إنشاء قاعدة البيانات (MySQL أو PostgreSQL) على السيرفر أو خدمة السحابة.
  2. تثبيت المشغّل إن لزم:
     - MySQL: `pip install pymysql` ثم استخدام:  
       `DATABASE_URL=mysql+pymysql://USER:PASSWORD@HOST:3306/DB_NAME`
     - PostgreSQL: `pip install psycopg2-binary` ثم استخدام:  
       `DATABASE_URL=postgresql://USER:PASSWORD@HOST:5432/DB_NAME`
  3. إنشاء ملف **`.env`** في مجلد المشروع (أو تعيين متغير البيئة على السيرفر) وتحديد:
     - `DATABASE_URL=...` فقط (بدون تغيير كود التطبيق).
  4. تشغيل الهجرات/الجداول على القاعدة الجديدة (نفس نماذج SQLAlchemy الحالية).

لا حاجة لتعديل الكود لتبديل القاعدة؛ تغيير `DATABASE_URL` كافٍ.

## 6. ملخص سريع

| الميزة | الحالة | أين / كيف التجربة |
|--------|--------|-------------------|
| إصلاح Hotspots (نقر → pitch/yaw + تطبيع سيرفر) | ✅ منفذ | إضافة نقطة من محرر النقاط ثم عرض البانوراما واللف |
| المفضلة (حفظ بانوراما 360) | ✅ تعمل | بانوراما → زر المفضلة → صفحة /favorites |
| بلاغ من نقطة + إرفاق صورة | ✅ تعمل | بانوراما → نقاط حمراء → بلاغ → نوع + ملاحظات + صورة → إرسال |
| ظهور البلاغ بعد الإرسال | ✅ | /issues و /received-issues وتفاصيل البلاغ |
| شات بوت + تعرف صوت (بدون كتابة) + TTS | ✅ تعمل | /trips-ai/chatbot → زر المايكروفون → تحدث → رد صوتي |
| إرشاد مكاني (خرائط + مسارات) | ✅ مكتمل | زر "إرشاد مكاني ذكي" → اكتب أو قل الوجهة (مثل المسجد الحرام، جمرة العقبة) → خريطة مصغّرة + رابط "فتح الاتجاهات في الخريطة" (Google Maps). الجيو-كود عبر OpenStreetMap (Nominatim). |
| **الإرشاد المكاني الذكي** (خرائط + صوت + AI) | ✅ مكتمل | **`/trips-ai/smart-guide`** — خريطة Leaflet، تكلم أو اكتب الوجهة، مسارات مشاة OSRM، TTS للإرشادات. يستخدم OpenRouter أو AI/ML API لاستخراج الوجهة من النص. أضف `OPENROUTER_API_KEY` أو `AIML_API_KEY` في `.env` |
| تقارير AI Excel + تصور (أعمدة) | ✅ تعمل | /ai-reports → اختيار نوع → تحميل Excel |
| خرائط ذهنية حقيقية | ❌ غير منفذة | التصور الحالي: رسوم بيانية داخل Excel فقط |
| ربط MySQL/PostgreSQL | ✅ جاهز | تعيين DATABASE_URL في .env وتركيب المشغّل المناسب |

إذا أردت توسيع ميزة معينة (مثل إرشاد مكاني على خريطة أو خرائط ذهنية)، يمكن تحديدها في طلب لاحق لتنفيذ خطوة بخطوة.
